HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Items in brackets are tags, the tags and content between make up an element.
  • Semantic elements are important for increasing the frequency of your page showing in a search, as well as making the code and page easier to review and traverse
  • Headers range from h1 to h6 and determine the size of the text placed there, the h1 header should only be used once in a page
  • HTML is the foundation of our webpage where we add the content before giving it style with CSS and functionality with JS

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element
  • A color code can be done using a semantic term such as gray or blue, or using a specific color code.
  • The selector, placed before declarations determines what elements the terms will be applied to.
  • The declartions determine the styling used and the value of that style. i.e. {color: blue} means the styling used is color and the value is blue
  • An external style sheet is a sheet used to store CSS seperately from the HTML, it must be linked via a link in the HTML's head element

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git pull origin main: pulls the code from the main branch to your local desktop.
  • git push origin "branch name" pushes code from local desktop to remote code(github).
  • git add -A adds modifications we made on the current branch to the staging area. The -A means we are adding all changes.
  • git commit -m "message" commits our code and -m allows us to add a message to describe what we are committing.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array needs to be used to define the variables before any function can use them.
  • /*your message here*/ can be used to make code inactive or store a message that the computer will ignore, alternativelly // can be used if it fits on one line.
  • You must call a function by typing out the function name followed by () before it will activate in the console.
  • Array items or indiced items are numbered starting at 0 and going up one whole number each time.
  • The console.log command shows the code what message to show in the console when the conditions are met.
  • A string must always be encased in single quotes while a number or boolean statement(true/false) do not go in quotes.
  • An if loop will continue to run until a truthy statement is found and executed at which point the code will go back to the control flow state.
